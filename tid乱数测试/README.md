# tid乱数测试

这是一个基于伊机脚本引擎的 FRLG TID 测试小项目。

当前版本已经按新脚本引擎的 `IMPORT` 规则做了模块化拆分：
- 主脚本只保留参数、公共变量和主流程
- `lib/*.ecs` 中存放具体函数
- `lib/ImgLabel/` 中存放本项目专用搜图标签

## 目录结构

```text
tid乱数测试/
├─ tid乱数测试.txt      # 主脚本入口
├─ README.md           # 项目说明
└─ lib/
   ├─ flow.ecs         # 流程辅助函数（如关闭软件）
   ├─ id.ecs           # 识图、TID匹配、结果打印
   ├─ name.ecs         # 取名相关函数
   ├─ timing.ecs       # 高精度计时函数
   └─ ImgLabel/        # 本项目使用的搜图标签
```

## 使用方式

1. 在伊机中打开 `tid乱数测试/tid乱数测试.txt`
2. 确认 `lib/` 和 `lib/ImgLabel/` 与主脚本位于同一项目目录下
3. 按需要修改主脚本顶部的参数
4. 从用户选择界面启动脚本

## 主脚本可调参数

以下参数都在 [tid乱数测试.txt](./tid乱数测试.txt) 顶部，通常只需要改这些。

### 目标相关

- `_TARGET_TID`
  目标 TID。

- `$same_id_switch`
  豹子号检测开关。
  `1` 为开启，`0` 为关闭。

- `$continue_id_switch`
  连号检测开关。
  `1` 为开启，`0` 为关闭。

- `$65535开关`
  额外匹配 `65535` 的开关。
  `1` 为开启，`0` 为关闭。

### 取名相关

- `$KeyDelay`
  取名输入时的按键间隔。

- `$Name1 ~ $Name7`
  主角名字的每一个字符。
  默认值为：

```text
A l a x e
```

如果某一位想保留为默认 `RED` 路线，可以填 `$RED`。

### 计时相关

- `$总循环次数`
  主循环执行次数。

- `_SEED_TIME`
  启动到第一次命中时间点的等待时间。

- `_PARITY_TIME`
  第二阶段修正奇偶时使用的等待时间。

- `_F1`
- `_F2`
- `_F3`
  后续三个关键阶段的时间参数。

这些值就是本脚本最核心的调参区。

## 名字编码写法

名字字符常量放在主脚本顶部，例如：

```text
$U_A
$U_B
$L_a
$L_b
$N_0
$M_Period
```

命名规则：
- `$U_` 开头：大写字母
- `$L_` 开头：小写字母
- `$N_` 开头：数字
- `$M_` 开头：符号

示例，名字 `Ben`：

```text
$Name1 = $U_B
$Name2 = $L_e
$Name3 = $L_n
$Name4 = $RED
$Name5 = $RED
$Name6 = $RED
$Name7 = $RED
```

## 模块说明

### `lib/id.ecs`

包含以下函数：
- `识图`
- `匹配`
- `打印参数`

负责读取 ID 卡数字、判断是否命中目标 TID、输出结果。

### `lib/name.ecs`

包含以下函数：
- `取名`
- `取名RED`
- `自定义名称`
- `向左移动光标`
- `向上移动光标`

负责主角取名流程。

### `lib/timing.ecs`

包含：
- `PreciseWait`

用于高精度等待。

### `lib/flow.ecs`

包含：
- `关闭软件`

负责循环开始前的关闭与回到启动流程。

## 注意事项

- `IMPORT "xxx.ecs"` 会从主脚本同目录下的 `lib/` 中加载对应文件，所以不要改乱目录层级。
- 本项目的搜图标签放在 `tid乱数测试/lib/ImgLabel/` 下，确保和脚本一起移动。
- 如果导入后报“脚本主语句只能存在一个文件中”，优先检查：
  - `.ecs` 文件里是否误放了主流程语句
  - `IMPORT` 的文件名是否和 `lib` 内文件一致
  - 项目目录是否被放错层级

## 后续维护建议

如果以后继续扩展这个项目，建议保持下面的约定：

- 主脚本只放：
  - `IMPORT`
  - 用户可调参数
  - 公共变量
  - 主流程

- `lib/*.ecs` 只放具体功能函数

这样后续继续加功能时，主脚本长度会比较稳定，也更方便排查问题。
